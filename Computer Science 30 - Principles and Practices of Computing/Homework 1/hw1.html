Problem #1: Python Functions

In this problem you will implement several Python functions. I've provided the hw1.py file to get you started, but there isn't much there! Do not use any language features of Python that have not been covered in class. Do not use any Python libraries other than the math library.
  1. Implement a function totalSeconds that takes 3 integers, in order representing a number of seconds, minutes, and hours. The function should return the total number of seconds that this represents. For example, totalSeconds(3,2,1) evaluates to 3723. Currently the hw1.py file contains an implementation of this function but it just returns 0 -- you should edit it to do the right thing.

  2. Implement a function money that takes a bank account balance (a number), an interest rate percentage per year (a number), and a number of years (a nonnegative integer). The function should return the amount of money that is in the bank account after the given number of years. You should assume that the interest is compounded, as is standard in most banks: the interest for one year becomes part of the principal and so itself accrues interest in future years. Also, you should round the final result to the nearest cent using Python's round function. For example, money(100, 7, 2) returns 114.49. Note: Do not do any rounding until the very end.

  3. Implement a function distanceFromNearestSquare that takes a positive integer as an argument and returns its distance from the nearest integer (either above or below it) that is a square. For example, distanceFromNearestSquare(13) returns 3 since 13 is 3 away from 16, which is a square. Hint: Python's math library will be useful. Also recall the built-in min function, which returns the minimum of its argument numbers.

  4. Implement a function isPrefix that takes two strings and returns True if either one of them is a prefix of the other, and False otherwise. For example, isPrefix('hel', 'hello') returns True, and so do isPrefix('hello', 'hel') and isPrefix('hello', 'hello'). Hint: The built-in min and len functions will be useful, as will the slicing operation for strings and the == operator for testing equality.

  5. Implement a function fromTotalSeconds that takes an integer argument representing a number of seconds and returns a list containing the corresponding number of seconds, minutes, and hours (in that order). The number of seconds in the result should always be less than 60, and similarly for the number of minutes. For example, fromTotalSeconds(3723) returns [3, 2, 1]. Hint: The modulus operator % will be useful, and so will the integer division operator //, which performs division but only returns the integer part of the result.

Problem #2: Picobot

Note: Your Picobot programs should work no matter where the robot starts. So make sure to test your programs starting from all sides and corners. You can use the Teleport Robot buttons on the bottom left of the Picobot playground to do that easily. We will be testing all of these starting points as part of our grading.

The Picobot playground is here.

  1. Write a Picobot program (i.e., a set of rules) that will cause the robot to traverse every cell in the rectangular world. Advice: Break the overall goal down into smaller tasks. Then write your rules incrementally to solve these tasks one by one. That is, write a few rules. See what they do. Then add new rules or modify the rules to either help address the current task or to move on to the next task. Continue in this way until you have a full solution.

In particular, copying and pasting the entire contents of this file back into the Picobot Rules window should result in a valid solution to the problem. Try doing that and make sure it works!
