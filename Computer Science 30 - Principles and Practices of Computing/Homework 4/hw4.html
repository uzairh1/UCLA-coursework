Problem 1: Warmup

  1. Computers don't know anything about triangles, but we can represent triangles with numbers, which is something that computers do know about. Suppose that we represent a triangle by a list of three positive integers, representing the three side lengths, which you can assume will always be in order from least to greatest. For example, [5, 8, 11] represents a triangle whose side lengths are respectively 5, 8, and 11. Implement a function rightTriangles that takes a list of triangles in the above representation and returns a list containing all, and only, the right triangles. Recall that a right triangle is one whose sides form a Pythagorean triple. For example, rightTriangles([[2, 3, 4], [3, 4, 5], [5, 12, 13], [8, 10, 12]]) returns [[3, 4, 5], [5, 12, 13]].

Problem 2: Text Manipulation

Computers don't know anything about characters or strings. Then how is Python able to support them? Through an encoding, in which each character is represented by a particular number. For example, internally the character 'a' is represented by the number 97. Python's ord function returns the encoding of a given character. For example, ord('a') returns 97. Python's chr function does this inverse: chr(97) returns 'a'.

The encodings of the lowercase letters between 'a' and 'z' are the 26 numbers between 97 and 122, inclusive. For example, the encoding of the letter 'e' is 101. The encodings of the uppercase letters between 'A' and 'Z' are the 26 numbers between 65 and 90, inclusive.

Below you'll implement several functions that manipulate strings. Each function is allowed to, and encouraged to, make use of the functions defined earlier.

  1. Implement a function toUpper that takes a single character as an argument and returns its uppercase version. If the character is already uppercase then simply return it. For example, toUpper('b') returns 'B' and so does toUpper('B'). You may assume that the given character is one of the 26 lowercase or 26 uppercase letters. Hint: No mapping, filtering, or reducing is required for this problem! The ord and chr functions will be useful.

  2. Now similarly implement a function toLower that takes a single character as an argument and returns its lowercase version. If the character is already lowercase then simply return it. For example, toLower('b') returns 'b' and so does toLower('B'). You may assume that the given character is one of the 26 lowercase or 26 uppercase letters.

  3. Implement a function allLower that takes a string and returns a version of the string where all letters are in lowercase. For example, allLower('hElLO') returns 'hello' and allLower('') returns ''. You may assume that the given string contains only characters representing English letters. Note: Because a string is a kind of sequence, you can use functions like map and filter to traverse a string one character at a time, which will be useful here. However, those functions will return lists of characters, not strings, so you'll also need to use map, filter, and/or reduce to turn a list of characters back into a string.

  4. Implement a function capitalize, which takes a string and returns a version of the string where the first letter is capitalized and the rest are in lowercase. For example, capitalize('hElLO') returns 'Hello'. You may assume that the given string contains only characters representing English letters.

  5. Finally, implement a function title, which takes a list of strings and capitalizes them appropriately as a book or movie title. Specifically, the first word should always be capitalized, any word whose length is at least 4 should also be capitalized, and the other words should be in lowercase. For example, title(['tHe', 'souND', 'AND', 'the', 'fUrY']) returns ['The', 'Sound', 'and', 'the', 'Fury']. You may assume that each string in the list contains only characters representing English letters.
