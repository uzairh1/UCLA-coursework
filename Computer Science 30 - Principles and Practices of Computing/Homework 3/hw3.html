We are living in the era of "big data." It's easier than ever for individuals, corporations, scientists, and governments to collect large amounts of data about themselves, society, and our world. But how do we make sense of all that data, and turn it into useful information? There are many different kinds of data analyses, and new approaches are being developed all the time.

Problem #1: Counting Data

One common data analysis task is counting, which is the task of computing the number of distinct items in a data set. For example, a text analysis may need to determine the number of distinct words in a given book, and an analysis of social media may need to determine the number of distinct commenters on a given post. In this problem you will implement a function to count the number of distinct elements in a list.
  1. First implement a function remove that takes an integer n and an integer list l. The function should produce a new list that represents the result of removing all occurrences of n in l. For example, remove(4, [-2, 3, 4, 0, 4, 1]) returns [-2, 3, 0, 1].

  2. Now implement a function countDistinct that accepts an integer list l and returns the number of distinct integers in the list. For example, countDistinct([5,3,2,2,5,1,3,2]) returns 4. There is a natural recursive formulation of this problem that uses the remove function you wrote above. In particular, the number of distinct elements in a non-empty list l is one more than the number of distinct elements in a version of l where all occurrences of its head are removed. For example, countDistinct([5,3,2,2,5,1,3,2]) is equal to 1 + countDistinct([3,2,2,1,3,2]).

Problem #2: Sorting Data

Another very common data analysis tasks is sorting, which is the task of ordering the data according to some criterion. For example, a social media site may want to order the posts that you are shown from most to least recent or from most to least popular, and an evolutionary biologist may want to order the observed mutations in a large sample of human DNA from most to least prevalent. There are many different algorithms for sorting, with different tradeoffs. The Week 4 discussion problem set guides you to implement selection sort. In this homework you will implement two other sorting algorithms, insertion sort and merge sort.

  1. First, implement a function insert that takes an integer n and an integer list l, where the elements of l can be assumed to be in sorted order from least to greatest. The function should produce a new list that represents the result of inserting n into l while maintaining sorted order. Note that duplicate elements are allowed. For example, insert(4, [2,3,3,5]) returns [2,3,3,4,5] and insert(2, [2,3,3,5]) returns [2,2,3,3,5]. Hint: Use our regular list recipe for recursion! In the recursive case, consider two different scenarios, one where n is less than or equal to the head of the list, and the other where n is greater than the head of the list.

  2. Implement a function insertionSort that takes a list of numbers and returns a list containing the same elements but sorted from least to greatest, using the insertion sort algorithm. For example, insertionSort([1,5,3,2,2,1,3,2]) returns [1,1,2,2,2,3,3,5]. Insertion sort is naturally recursive and conceptually works as follows. If the list is empty then it's already sorted. Otherwise, recursively sort the tail of the list. Finally, insert the head of the list into that recursive result, using your insert function above.

  3. Write a function merge that takes two integer lists l1 and l2 as arguments, which you can assume are each in sorted order from least to greatest. The function should produce a new list that contains the elements of both input lists, also sorted from least to greatest. Note that duplicates are allowed, both within and between lists. For example, merge([1,3,5], [2,2,3,4]) returns [1,2,2,3,3,4,5]. Your solution should not call any functions other than merge itself. It is tempting to use the insert function above to implement merge, but that is not allowed (and will result in a much less efficient solution). Instead, there is a natural recursive approach that you should employ. In our example above, for instance, merge([1,3,5], [2,2,3,4]) is equal to [1] + merge([3,5], [2,2,3,4]).

  4. Now implement a function mergeSort takes an integer list and returns a list containing the same elements but sorted from least to greatest, using the merge sort algorithm. For example, mergeSort([1,5,3,2,2,1,3,2]) returns [1,1,2,2,2,3,3,5]. The merge sort algorithm conceptually works as follows. First, use slicing to split the list into two equal (or in the case of an odd number of elements, close to equal) size lists --- in our example this produces [1,5,3,2] and [2,1,3,2]. Then sort each of these lists by calling mergeSort recursively --- in our example this produces [1,2,3,5] and [1,2,2,3]. Finally use your merge function from above to merge the resulting lists to obtain the full sorted list. You can see an animation of merge sort in action on the right-hand side of the page here.
